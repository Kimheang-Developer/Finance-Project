거래처에 지급 건이 많으신가요?
ERP에서 확정한 지급내역을 바로 이체 할 수 있어 안전하고 편리합니다.

연계효과

  • ERP에서 지출결의를 완료한 지급내역을 바로 이체 할 수 있어 안전하고 편리합니다.

    복잡한 파일 등록, 변환 등의 수작업 없이 지급파일을 은행으로 전송할 수 있어 안전하고 빠르게 대금지급 업무를 처리할 수 있습니다.

  • 자금사고 예방을 위한 내부통제 강화를 기대할 수 있습니다.

    내부 시스템에서 지급승인이 완료된 데이터는 임의로 삭제 및 변경 할 수 없습니다.
    지급 실행 시 은행 기업인터넷뱅킹 수준의 다단계결재, 인증서 제출 등의 내부통제 과정을 적용하여 자금사고를 예방 할 수 있습니다.

연계사례

  • 더존비즈온

    “대금지급 연계를 통한자금 사고 최소화”

      • 고객명: N사
      • 업종: 서비스업
      • 매출액: 100억 이상

    ERP에서 생성된 지급파일을 브랜치에서 한번 더 확인하여 위변조 또는 오입력의 가능성을 최소화하고,
    다단계 결재를 통한 지급으로 내부 통제를 강화

  • 자체구축

    “보안성과 효율성을 강화한 지급업무 자동화”

      • 고객명: I건설
      • 업종: 건설업
      • 매출액: 200억 이상

    월 400건 정도의 지급 건을 인터넷뱅킹의 건별 이체를 통해 수작업으로 직접 이체 하였으나, ERP에서 작성한 ‘지출 데이터’를
    브랜치로 전송하여 지급함으로써 지급업무의 보안성이 강화하고 담당자의 업무 효율성을 향상
